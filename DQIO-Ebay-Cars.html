Initial observation¶

Looking at the first 5 rows: Price could do with being turned into a numeric data type odometer should be changed into numeric dtype notRepairedDamage could be turned into bit abtest could be turned into bit

Of the 5000 rows of data in this subset, we have multiple columns with missing data, including: vehicleType, model, fuelType, notRepairedDamage. It could be possible that there are other rows with the same vehicle brand and model that can help us populate the vehicleType and possibly fuelType. It is also possible to use name to populate missing data in model

Clearing up the price column¶

Looking at the length of price values, the max price length is 11. Although this alone isn't a problem or odd, when we look at all the rows with values who price length is over the length of 10 (this is inclusive of , and $) we can see some patterns of possible incorrect data where users have typed in 11,111,111 and 123,456,78 for example. I have a high confidence that these values are likely poor/unuseful data.

def brandLoop(data):
    '''
        Function is to calculate the average price per brand of autos df.
        data = two columns: brands, price
    '''
    brands_average = {}
    
    for brand in data:
        if brand not in brands_average.keys():
            brands_average[brand] = autos[autos['brand'] == brand]['price'].sum() / autos[autos['brand'] == 'peugeot']['price'].count()
    
    return brands_average
